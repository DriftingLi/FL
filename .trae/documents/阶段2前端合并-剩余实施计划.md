## 转换规则（Ant Design Vue → Element Plus，机械转换）

通用映射（已与 6 个已完成 component 风格一致）：
| Ant Design Vue | Element Plus |
|---|---|
| `v-model:value="x"` | `v-model="x"` |
| `<a-button :icon="h(XxxOutlined)" type="primary" :loading>` | `<el-button :icon="Xxx" type="primary" :loading>`（icon 传组件引用，非 h()） |
| `<a-form-item label required>` | `<el-form-item label required>`（裸用，无 el-form 包裹，与源一致） |
| `<a-row :gutter>` / `<a-col :xs :md :lg>` | `<el-row :gutter>` / `<el-col :xs :md :lg>`（24 栅格一致） |
| `<a-input v-model:value>` | `<el-input v-model>` |
| `<a-input-number v-model:value :min :max :step :precision>` | `<el-input-number v-model :min :max :step :precision>` |
| `<a-textarea v-model:value :rows>` | `<el-input v-model type="textarea" :rows>` |
| `<a-select v-model:value show-search :filter-option :options :disabled allow-clear>` | `<el-select v-model filterable :disabled clearable>` + 内联 `<el-option v-for>`（见 D5） |
| `<a-radio-group v-model:value>` / `<a-radio :value>` | `<el-radio-group v-model>` / `<el-radio :value>` |
| `<a-spin :spinning>` | `<div v-loading="...">` 包裹（见 D4） |
| `<a-empty description>` | `<el-empty description>` |
| `<a-collapse :bordered="false">` + `<a-collapse-panel :header :key>` | `<el-collapse v-model>` + `<el-collapse-item :name :title>`（见 D6） |
| `<a-table :data-source :columns :row-key :pagination="false" :scroll size>` + `#bodyCell` slot | `<el-table :data row-key size>` + 内联 `<el-table-column>` + `#default="{ row }"` scoped slot（见 D8） |
| `<a-descriptions :column bordered size>` + `<a-descriptions-item label>` | `<el-descriptions :column border size>` + `<el-descriptions-item label>`（`bordered`→`border`） |
| `<a-list :data-source>` + `#renderItem` + `<a-list-item-meta>` | 自定义 `<div v-for>` 循环（见 D9） |
| `<a-tag :color>` | `<el-tag effect="plain" :style="{color,borderColor}">`（见 D7） |
| `<a-progress :percent :stroke-color :show-info :size>` | `<el-progress :percentage :color :show-text :stroke-width>` |
| `<a-upload :before-upload :show-upload-list accept>` | `<el-upload :before-upload :show-file-list accept>` |
| `import { message } from 'ant-design-vue'` + `message.error/success` | `import { ElMessage } from 'element-plus'` + `ElMessage.error/success` |
| `@ant-design/icons-vue` 按钮图标 | `@element-plus/icons-vue` 组件引用（见 D3） |
| `@ant-design/icons-vue` 主题/标题图标 | emoji `<span>`（见 D3） |

## 假设与决策（执行者无需再决策）

- **D1 路由迁移**：所有硬编码路由迁移如下（必须逐处替换）：
  - `useEvaluationForm.ts:115` `router.push({ name: 'result' })` → `router.push({ name: 'ValuationResult' })`
  - view 内 `router.push('/input/electric')` → `/valuation/input/electric`
  - `router.push('/input/combustion')` → `/valuation/input/combustion`
  - `router.push('/result')` → `/valuation/result`
  - `router.push('/report/${id}')` → `/valuation/report/${id.value}`
  - `router.push('/battery/result')` → `/valuation/battery/result`
  - `router.push('/battery')` → `/valuation/battery`
  - `router.replace('/battery')` → `/valuation/battery`
  - `router.replace('/')`（ResultView 守卫）→ `/valuation`
- **D2 PageHeader `#icon` slot 删除**：源 view 的 `<template #icon>...</template>` 是死代码（PageHeader 无此 slot，仅有 `icon` prop）。转换时删除该 slot 块，不新增 `:icon` prop（保持原视觉无图标，最小改动）。
- **D3 图标策略**：
  - el-button `:icon` 用 Element Plus 组件引用：`Refresh`(重置)、`Promotion`(提交/开始评估)、`CircleCheck`(生成PDF)、`Edit`(返回修改)、`Document`(查看报告)、`Download`(下载PDF)、`ArrowLeft`(返回结果)、`Upload`(上传json)、`ArrowRight`(HomeView CTA)。从 `@element-plus/icons-vue` 命名导入。
  - 主题/标题内联图标用 emoji `<span>`：⚡(电动)、🔥(内燃)、💡(建议)、→(CTA 箭头)。HomeView entry-card-icon 用 emoji span。
  - 转换后若 `h` 不再被使用则移除 `h` import（ReportView/ResultView/BatteryResultView/ElectricInputView/CombustionInputView 均需检查）。
- **D4 `a-spin` → `v-loading`**：用 `<div v-loading="loadingX" class="...">` 包裹原 spin 内容。ElectricInputView/CombustionInputView 用 `v-loading="loadingConfigs"`；ReportView 用 `v-loading="loading"`。
- **D5 `a-select` 搜索**：`show-search :filter-option="filterBrandOption"` → 仅 `filterable`（Element 内置按 label 子串匹配，等价于 `filterBrandOption`）。删除 view 中的 `filterBrandOption` 函数（不再需要）。`:options` → 内联 `<el-option v-for>`。
- **D6 `el-collapse` 需 v-model**：Element el-collapse 无内部态，必须 `v-model`。ConditionTable：`const activeNames = ref<string[]>([])`，watch configs 首次加载时设 `activeNames.value = configs.length<=4 ? configs.map(c=>c.category_code) : []`（复刻 `default-active-key` 语义）。ReportView：`const reportActiveNames = ref<string[]>([])`（默认全收起，可手动展开）。`a-collapse-panel :header="X"` → `el-collapse-item :title="X" :name="code"`。
- **D7 `a-tag :color` → `el-tag`**：用 `<el-tag effect="plain" :style="{ color: STATUS_COLOR[row.status], borderColor: STATUS_COLOR[row.status] }">`（可靠跨版本着色）。ReportView 状态列用此。
- **D8 `a-table` + `#bodyCell`/`customRender` → `el-table` + 内联 `el-table-column`**：
  - ConditionTable：`columns` 数组与 `TableColumnsType` import 删除；改为 `<el-table :data="cat.items" row-key="item_code" size="default">` + `<el-table-column prop="item_name" label="部件条目">` + `<el-table-column label="状态" width="280"><template #default="{ row }">...el-select...</template></el-table-column>`。
  - ReportView：`reportItemColumns` 数组、`getCategoryItemRowKey`、`AlignType` 类型全部删除；改为内联 `<el-table-column>` + `#default="{ row }"` scoped slot（权重/评分用 `<span class="num">`，状态用 D7 的 el-tag）。`row-key="id"`。
- **D9 `a-list` → 自定义循环**：BatteryResultView 的 Top-5 特征列表改为 `<div v-for="(item,index) in topFeatures" :key="index" class="feature-item">` + 内部 `<span class="feature-rank">`/`<span class="feature-name">`/`<el-tag>`/`<el-progress :percentage :color :show-text="false" :stroke-width="6">`。
- **D10 `valuation-root` 作用域类**：每个 view 根 `<div>` 加 `valuation-root` 类（与已有 6 component 的 token 作用域约定一致），确保 valuation-tokens.css 的 CSS 变量在该视图生效。ConditionTable 是 component，由父 view 提供作用域，无需自身加类（其 `<style scoped>` 已用 `var(--...)`，变量从父级 DOM 继承解析）。
- **D11 菜单图标**：DefaultLayout 新增菜单项用 `DataAnalysis`（`@element-plus/icons-vue`，残值评估语义契合；若该名不存在回退 `Coin`）。
- **D12 `el-select @change`**：Element el-select `@change` 直接 emit value；ConditionTable 保留 `onStatusSelectChange` 做 narrow 守卫（与源一致，安全）。
- **D13 不碰已确认完成的文件**：6 个已转换 component、client.ts、valuation-tokens.css、18 个 .ts 不重写（除非 D1 要求的 useEvaluationForm.ts 单行修改）。
- **D14 不删除残值评估/frontend 源目录**（计划约束）。
- **D15 build 脚本**：`package.json` 已是 `"build": "vite build"`（解耦 type-check），保持不变。
